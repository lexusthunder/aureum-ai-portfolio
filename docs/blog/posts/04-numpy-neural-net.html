I wrote a neural network in NumPy. You should too.

Not for production. For interviews. For first principles. For the moment when a senior engineer asks "what's actually happening on the backward pass" and you can answer without reading the docs.

The temptation, when learning ML, is to start with PyTorch. torch.nn.Linear, .backward(), an optimizer, a fit loop, and you're shipping in an afternoon.

I did this for a year. It worked. I built models. I deployed them. I won small Kaggle competitions.

Then a senior engineer at a Series A asked me, in a 1:1 on a take-home: "What does .backward() actually compute?"

I said something about gradients. He asked me to write it. Without PyTorch.

I couldn't. Not in 30 minutes. Not in 2 hours. I could write the forward pass — that's just matrix multiplication. The backward pass, I had only ever read about. I had never derived it.

I went home that weekend and wrote one.

Why this matters more than you think

Three reasons.

1. Senior engineers test fundamentals

Mid-level interviews ask you to use the framework. Senior interviews ask you to be the framework. The signal a senior is looking for: can this person debug my training run when PyTorch's autograd does something surprising?

If you've never written backprop yourself, the answer is no. You'll google. The senior knows.

2. It's a load-bearing piece of intuition

Once you've written dL/dW = X.T @ dL/dY with your own hands, you understand why batch size affects gradient noise, why weight initialization matters, why ReLU dying is a real thing. Before that, those are facts you memorised. After, they're obvious.

3. It de-risks LLMs for you

If you can write a 2-layer NN in NumPy, then a transformer is just (a) more layers, (b) a fancy attention computation, (c) some normalisation. The entire LLM stack becomes legible.

The exact code I wrote

This is the heart of b2b-lead-scoring-api on my portfolio. ~80 lines, no deps but NumPy:

import numpy as np

class B2BLeadScoringModel:
    """2-layer feedforward NN with sigmoid output, hand-written backprop."""

    def __init__(self, n_features=5, n_hidden=8, lr=0.05, seed=42):
        rng = np.random.default_rng(seed)
        # He initialization for ReLU
        self.W1 = rng.standard_normal((n_features, n_hidden)) * np.sqrt(2 / n_features)
        self.b1 = np.zeros(n_hidden)
        self.W2 = rng.standard_normal((n_hidden, 1)) * np.sqrt(2 / n_hidden)
        self.b2 = np.zeros(1)
        self.lr = lr

    def _sigmoid(self, z):  # numerically stable
        return np.where(z >= 0, 1 / (1 + np.exp(-z)), np.exp(z) / (1 + np.exp(z)))

    def _relu(self, z):
        return np.maximum(0, z)

    def forward(self, X):
        self.X = X
        self.Z1 = X @ self.W1 + self.b1
        self.A1 = self._relu(self.Z1)
        self.Z2 = self.A1 @ self.W2 + self.b2
        self.A2 = self._sigmoid(self.Z2)
        return self.A2

    def backward(self, y_true):
        m = self.X.shape[0]
        # Loss: binary cross-entropy
        # dL/dZ2 for sigmoid + BCE simplifies to (A2 - y) / m
        dZ2 = (self.A2 - y_true.reshape(-1, 1)) / m
        dW2 = self.A1.T @ dZ2
        db2 = dZ2.sum(axis=0)
        dA1 = dZ2 @ self.W2.T
        dZ1 = dA1 * (self.Z1 > 0)  # ReLU gradient
        dW1 = self.X.T @ dZ1
        db1 = dZ1.sum(axis=0)
        # SGD update
        self.W1 -= self.lr * dW1
        self.b1 -= self.lr * db1
        self.W2 -= self.lr * dW2
        self.b2 -= self.lr * db2

    def fit(self, X, y, epochs=2000):
        for _ in range(epochs):
            self.forward(X)
            self.backward(y)
        return self

    def predict_proba(self, X):
        return self.forward(X).ravel()

The three things to internalise

(1) The gradient of (sigmoid + BCE) is just (predicted − true)

This is the magic moment. The math:

L = -[y·log(σ(z)) + (1-y)·log(1-σ(z))]
dL/dz = σ(z) − y
      = predicted − target

That's it. That's the entire reason logistic regression works the way it does. The output error is the gradient. There is no chain rule trick — sigmoid and BCE are designed to compose into this clean form.

(2) The chain rule is just matrix multiplication, in reverse

Forward: Z1 = X @ W1, A1 = ReLU(Z1), Z2 = A1 @ W2, A2 = sigmoid(Z2).

Backward (reverse the multiplications, transpose the matrices):

dZ2 = A2 - y
dW2 = A1.T @ dZ2     ← input was A1, so transpose A1
dA1 = dZ2 @ W2.T     ← we want gradient w.r.t. A1, so transpose W2
dZ1 = dA1 * (Z1 > 0) ← ReLU's gradient is 1 where Z>0, else 0
dW1 = X.T @ dZ1      ← input was X, so transpose X

If you stare at this until it clicks, you have backprop in your bones. Everything else (Adam, momentum, layer norm) is a refinement.

(3) The whole thing trains in 50 lines of NumPy

No autograd graph. No GPU. No batching machinery. Just `forward` and `backward` and a loop. This is the floor of what every ML library is doing under the hood.

What to build to internalise this

  1. Implement the snippet above on the Iris or Wisconsin breast cancer dataset. ~95% accuracy. Should run in 30 seconds.
  2. Add a third layer. Watch how the gradients chain.
  3. Replace BCE with cross-entropy + softmax. Derive dL/dZ from scratch. (Spoiler: same form, predicted − target.)
  4. Implement Adam from the paper. Two extra arrays per parameter (running average of gradient + running average of squared gradient).

~3 hours of work. It will pay you back for the rest of your career.

The interview test

Next time a senior asks "explain backprop", you don't quote a textbook. You ask for a whiteboard. You write the loss. You take the derivative. You match the matrix shapes. You explain why we transpose.

The senior smiles. You're hired.
